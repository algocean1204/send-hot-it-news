# 모듈 블럭 설계도 — news-pulse

## 프로젝트 개요

12개 IT/AI 소스에서 뉴스를 수집하고, 로컬 LLM으로 요약/번역한 뒤, 텔레그램으로 시간당 1회 푸시하는 봇이다.

- macOS 전용, 상시 프로세스 없음 (launchd 트리거)
- 추가 비용 0원 (로컬 LLM + Claude CLI 폴백)
- Python 3.12 + uv + Ollama + Flutter macOS (Riverpod) + SQLite

---

## 설계 원칙

- 1 블럭 = 1 Protocol(인터페이스). IN/OUT이 명확히 정의됨
- 메인 오케스트레이터가 블럭을 주입받아 조립
- 블럭 교체 = 같은 Protocol의 구현체로 교체, 오케스트레이터 한 줄만 수정
- 블럭끼리는 서로 모름. 공유 데이터 모델(dataclass)로만 통신
- Strategy Pattern + Dependency Injection 적용

## 🏗️ 조립 예시 (orchestrator.py)

```python
# 블럭 생성
config = ConfigLoader(".env").load()
engine = OllamaEngine(config)  # ← 엔진 교체 시 이 줄만 수정
db = SqliteStore(config.db_path)

# 파이프라인 조립
pipeline = Pipeline(
    poller=SubscriberPoller(config),
    fetchers=[RssFetcher(), AlgoliaFetcher(), RedditFetcher(), GithubAtomFetcher()],
    dedup=SqliteDedup(db),
    lang_detector=SourceFirstDetector(),
    filter=FilterChain([BlacklistFilter(), TierRouter(config), PrioritySelector(config)]),
    summarizer=FallbackChain([
        ApexSummarizer(engine),
        KananaSummarizer(engine),
        ClaudeCliSummarizer(ClaudeCliEngine()),
    ]),
    translator=FallbackChain([
        KananaTranslator(engine),
        ClaudeCliTranslator(ClaudeCliEngine()),
    ]),
    hot_detector=HotNewsDetector(config, db),
    formatter=MessageFormatter(),
    sender=TelegramSender(config),
    logger=RunLogger(db),
    notifier=ErrorNotifier(config),
)

pipeline.run()
```

## 각 블럭 상세 명세

### 블럭 1 — ConfigLoader

**핵심 책임**: .env 파일에서 시크릿/설정을 로드하고 Config 객체로 반환한다.

**Protocol 인터페이스**

```python
class ConfigLoader(Protocol):
    def load(self) -> Config: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| env_path | str | .env 파일 경로 (기본: 프로젝트 루트) |

**OUT**

| 필드 | 타입 | 설명 |
|------|------|------|
| Config | dataclass | 봇 토큰, DB 경로, 소스 설정, 모델 설정 등 전체 설정 |

**구현체**

없음. 단일 구현. dotenv 계열 라이브러리로 .env 파싱 후 Config dataclass로 변환한다.

**의존성**: 없음 (파이프라인 첫 번째 블럭)

**특이사항**

- TZ 이중 고정: `os.environ['TZ'] = 'Asia/Seoul'` + `time.tzset()` 호출
- 필수 키 누락 시 즉시 ValueError 발생, 빈 문자열도 누락으로 처리

**에러 처리**

- 필수 환경 변수 누락 → ValueError 발생, 파이프라인 즉시 중단

---

### 블럭 2 — MemoryGuard

**핵심 책임**: 시스템 가용 메모리를 확인하고, 로컬 LLM 사용 가능 여부를 판단한다.

**Protocol 인터페이스**

```python
class MemoryGuard(Protocol):
    def check(self, config: Config) -> MemoryStatus: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| config | Config | 메모리 임계값 설정 포함 |

**OUT**

| 값 | 타입 | 조건 |
|----|------|------|
| "local_llm" | MemoryStatus | 가용 RAM ≥ 26GB |
| "claude_fallback" | MemoryStatus | 가용 RAM < 26GB |

**구현체**

없음. 단일 구현. `psutil.virtual_memory().available`로 가용 메모리 측정.

**의존성**: ConfigLoader (임계값 설정 읽기)

**에러 처리**

- psutil 호출 실패 → 안전하게 "claude_fallback" 반환 (로컬 LLM 시도 안 함)

---

### 블럭 3 — SubscriberPoller

**핵심 책임**: 텔레그램 getUpdates로 /start 명령어를 확인하고, 신규 구독 요청을 DB에 저장한다.

**Protocol 인터페이스**

```python
class SubscriberPoller(Protocol):
    def poll(self, config: Config) -> list[SubscriberEvent]: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| config | Config | 봇 토큰 포함 |

**OUT**

| 타입 | 설명 |
|------|------|
| List[SubscriberEvent] | 이번 실행에서 감지된 구독 이벤트 목록. subscribers 테이블에 저장됨 |

**구현체**

없음. 단일 구현. Telegram Bot API `getUpdates` 엔드포인트 호출.

**의존성**: ConfigLoader

**에러 처리**

- API 호출 실패 → 빈 리스트 반환, 파이프라인 계속 진행
- 이미 처리된 update_id는 offset으로 건너뜀

### 블럭 4 — Fetcher

**핵심 책임**: 12개 소스에서 뉴스를 병렬 수집하고 RawItem으로 정규화한다.

**Protocol 인터페이스**

```python
class Fetcher(Protocol):
    def fetch(self, config: Config) -> list[RawItem]: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| config | Config | 소스 목록, ON/OFF 설정, 타임아웃 |

**OUT**

| 타입 | 설명 |
|------|------|
| List[RawItem] | 수집된 원시 뉴스 아이템. 중복 포함, 필터링 전 |

**구현체**

| 구현체 | 담당 소스 |
|--------|---------|
| RssFetcher | RSS/Atom 피드 소스 (대부분의 AI랩 블로그, GeekNews 등) |
| AlgoliaFetcher | Hacker News (algolia API, JSON) |
| RedditFetcher | Reddit 서브레딧 (.json 엔드포인트) |
| GithubAtomFetcher | GitHub Trending, 특정 저장소 릴리스 (Atom 피드) |

**병렬 수집**: `asyncio` 또는 `ThreadPoolExecutor` 사용. 소스별 독립 실행, 일부 실패해도 나머지 계속 수집.

**의존성**: ConfigLoader (소스 목록, ON/OFF)

**에러 처리**

- 개별 소스 실패 → 해당 소스 건너뜀, 나머지 수집 계속
- 전체 타임아웃 초과 → 수집된 것만 반환

---

### 블럭 5 — Dedup

**핵심 책임**: URL 해시 기반으로 이미 처리한 뉴스를 걸러내고 신규 아이템만 반환한다.

**Protocol 인터페이스**

```python
class Dedup(Protocol):
    def filter_new(self, items: list[RawItem]) -> list[NewsItem]: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| items | List[RawItem] | Fetcher 출력 원시 아이템 |

**OUT**

| 타입 | 설명 |
|------|------|
| List[NewsItem] | 중복 제거된 신규 아이템만. RawItem → NewsItem 변환 포함 |

**구현체**

| 구현체 | 설명 |
|--------|------|
| SqliteDedup | processed_items 테이블에서 URL 해시(SHA256) 비교. 신규 항목은 즉시 테이블에 삽입 |

**의존성**: ConfigLoader (DB 경로)

**에러 처리**

- DB 접근 실패 → 전체 아이템을 신규로 간주 (중복 허용, 안전한 방향 선택)

### 블럭 6 — LanguageDetector

**핵심 책임**: 뉴스 아이템의 언어를 감지하여 lang 필드를 추가한다.

**Protocol 인터페이스**

```python
class LanguageDetector(Protocol):
    def detect(self, item: NewsItem) -> NewsItem: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| item | NewsItem | lang 필드 없는 뉴스 아이템 |

**OUT**

| 타입 | 설명 |
|------|------|
| NewsItem | lang 필드 추가됨 ("ko" \| "en") |

**구현체**

| 구현체 | 전략 |
|--------|------|
| SourceFirstDetector | 소스 출처 기반 1차 판단. GeekNews → "ko" 고정, 나머지 11개 소스 → "en" 고정 |
| LinguaDetector | lingua 라이브러리 기반 폴백. 소스 기반 판단이 불확실할 때만 실행 |

두 구현체를 체인으로 연결: SourceFirstDetector 결과가 확실하면 LinguaDetector 호출 안 함.

**의존성**: 없음

**에러 처리**

- lingua 감지 실패 → "en" 기본값 사용

### 블럭 7 — Filter

**핵심 책임**: 계층화된 하이브리드 필터링으로 전송할 뉴스 최대 8건을 선별한다.

**Protocol 인터페이스**

```python
class Filter(Protocol):
    def apply(self, items: list[NewsItem], config: Config) -> list[NewsItem]: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| items | List[NewsItem] | 언어 감지 완료된 뉴스 아이템 |
| config | Config | Tier별 임계값, 블랙리스트 설정 |

**OUT**

| 타입 | 설명 |
|------|------|
| List[NewsItem] | 필터 통과 아이템. 최대 8건 |

**구현체 (체인 순서)**

| 순서 | 구현체 | 역할 |
|------|--------|------|
| 1 | BlacklistFilter | 키워드 블랙리스트 전체 적용. 매칭 아이템 즉시 제거 |
| 2 | TierRouter | 소스별 Tier 분류 및 할당량 적용 |
| 3 | PrioritySelector | Tier별 우선순위 정렬 후 최종 8건 선택 |

**Tier 정책**

| Tier | 소스 | 정책 |
|------|------|------|
| 1 | AI랩 블로그, GitHub Trending | 무조건 통과 (할당량 7건) |
| 2 | GeekNews | 큐레이션 신뢰, 무조건 통과 (할당량 1건) |
| 3 | Reddit, Hacker News | 업보트 임계값 + 게시 시간 연령 보정 적용 (할당량 4건) |
| 4 (전역) | 전체 | 키워드 블랙리스트 |

**의존성**: ConfigLoader (Tier 임계값, 블랙리스트)

**에러 처리**

- 필터 체인 중 예외 → 해당 필터 건너뜀, 다음 필터 계속 적용

### 블럭 8 — ModelEngine

**핵심 책임**: AI 모델을 로드/언로드하고 추론을 실행한다. 동시 적재 금지.

**Protocol 인터페이스**

```python
class ModelEngine(Protocol):
    def load(self, model_name: str, keep_alive: int = 0) -> None: ...
    def generate(self, prompt: str, options: dict) -> str: ...
    def unload(self, model_name: str) -> None: ...
    def is_available(self) -> bool: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| model_name | str | 모델 식별자 (예: "APEX-4B", "kanana-1.5-8b") |
| prompt | str | 완성된 프롬프트 문자열 |
| options | dict | temperature, max_tokens 등 생성 옵션 |

**OUT**

| 타입 | 설명 |
|------|------|
| str | 모델 응답 텍스트 |

**구현체**

| 구현체 | 방식 |
|--------|------|
| OllamaEngine | Ollama REST API 사용. `keep_alive=0`으로 생성 직후 즉시 언로드. load/generate/unload 순서 엄격히 준수 |
| ClaudeCliEngine | `claude -p --bare` subprocess 실행. load/unload 없이 generate만 구현. is_available()은 `which claude` 확인 |

**순차 로드 원칙**: APEX 사용 완료 후 언로드 확인 → Kanana 로드. 동시에 두 모델 메모리 적재 금지.

**의존성**: ConfigLoader (모델 설정, Ollama 엔드포인트)

**에러 처리**

- Ollama 연결 실패 → is_available() False 반환, Summarizer가 폴백 체인으로 처리
- subprocess 실패 → CalledProcessError 발생, Summarizer가 처리

---

### 블럭 9 — Summarizer

**핵심 책임**: 뉴스 아이템을 요약한다. 프롬프트 구성과 응답 파싱만 담당하고, 실제 추론은 ModelEngine에 위임한다.

**Protocol 인터페이스**

```python
class Summarizer(Protocol):
    def summarize(self, item: NewsItem, engine: ModelEngine) -> SummaryResult: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| item | NewsItem | 요약할 뉴스 아이템 |
| engine | ModelEngine | 주입된 모델 엔진 |

**OUT**

| 타입 | 설명 |
|------|------|
| SummaryResult | 요약 텍스트 + 메타 정보 |

**구현체**

| 구현체 | 사용 모델 | 특이사항 |
|--------|---------|---------|
| ApexSummarizer | APEX-4B (OllamaEngine) | 한국어/영어 모두 요약. 한국어 소스는 한국어로 직접 요약 |
| KananaSummarizer | kanana-1.5-8b (OllamaEngine) | APEX 폴백. 단독 요약 시 사용 |
| ClaudeCliSummarizer | claude CLI (ClaudeCliEngine) | 최종 폴백. subprocess 실행 |

**FallbackChain 순서**: ApexSummarizer → KananaSummarizer → ClaudeCliSummarizer

**의존성**: ModelEngine (주입)

**에러 처리**

- 요약 실패 (타임아웃, 파싱 오류) → FallbackChain이 다음 구현체 시도
- 전체 폴백 실패 → SummaryResult에 error 필드 표시, 해당 아이템 전송 스킵

---

### 블럭 10 — Translator

**핵심 책임**: 영어 요약 결과를 한국어로 번역한다. 프롬프트 구성과 응답 파싱만 담당하고, 실제 추론은 ModelEngine에 위임한다.

**Protocol 인터페이스**

```python
class Translator(Protocol):
    def translate(self, result: SummaryResult, engine: ModelEngine) -> SummaryResult: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| result | SummaryResult | 요약 완료 결과 (영어) |
| engine | ModelEngine | 주입된 모델 엔진 |

**OUT**

| 타입 | 설명 |
|------|------|
| SummaryResult | 한국어 번역이 적용된 요약 결과 |

**조건부 실행**: `item.lang == "ko"`이면 번역 스킵, 원본 SummaryResult 그대로 반환.

**구현체**

| 구현체 | 사용 모델 |
|--------|---------|
| KananaTranslator | kanana-1.5-8b (OllamaEngine). 한국어 특화 모델 |
| ClaudeCliTranslator | claude CLI (ClaudeCliEngine). 폴백 |

**FallbackChain 순서**: KananaTranslator → ClaudeCliTranslator

**의존성**: ModelEngine (주입)

**에러 처리**

- 번역 실패 → 영어 요약 원본 유지, error 플래그 표시

---

### 블럭 11 — HotNewsDetector

**핵심 책임**: 핫뉴스 여부를 판단하고 hot_news 테이블에 영구 저장한다.

**Protocol 인터페이스**

```python
class HotNewsDetector(Protocol):
    def detect(self, item: NewsItem, result: SummaryResult) -> bool: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| item | NewsItem | 뉴스 아이템 (업보트 수 포함) |
| result | SummaryResult | 요약 결과 |

**OUT**

| 타입 | 설명 |
|------|------|
| bool | True면 핫뉴스. hot_news 테이블에 저장됨 |

**구현체**

없음. 단일 구현.

**판단 기준**

**의존성**: ConfigLoader (임계값), DB (hot_news 저장)

### 블럭 12 — MessageFormatter

**핵심 책임**: 뉴스 아이템과 요약 결과를 텔레그램 MarkdownV2 형식의 메시지 문자열로 변환한다.

**Protocol 인터페이스**

```python
class MessageFormatter(Protocol):
    def format(self, item: NewsItem, result: SummaryResult, is_hot: bool) -> str: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| item | NewsItem | 뉴스 아이템 |
| result | SummaryResult | 한국어 요약 결과 |
| is_hot | bool | 핫뉴스 여부 |

**OUT**

| 타입 | 설명 |
|------|------|
| str | 텔레그램 MarkdownV2 포맷 문자열 |

**포맷 정책**

- 일반 뉴스: 제목 + 한줄 요약 + 링크. 간결 기술체
- 핫뉴스: 제목 + 상세 요약 + 주요 내용 + 링크. 더 많은 맥락 포함
- 이모지 + 구분선(═══) + 뉴스 간 2줄 공백
- MarkdownV2 특수문자 이스케이프 처리 필수

**구현체**

없음. 단일 구현.

**의존성**: 없음

**에러 처리**

- 포맷 실패 → 플레인 텍스트 폴백 (이스케이프 없이 제목 + 링크만)

### 블럭 13 — TelegramSender

**핵심 책임**: 포맷된 메시지를 관리자와 승인된 구독자에게 전송한다.

**Protocol 인터페이스**

```python
class TelegramSender(Protocol):
    def send(self, message: str, chat_ids: list[str]) -> SendResult: ...
```

**IN**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| message | str | 텔레그램 MarkdownV2 메시지 |
| chat_ids | List[str] | 전송 대상 chat_id 목록 |

**OUT**

| 타입 | 설명 |
|------|------|

**구현체**


## ModelEngine 구조 상세

```
ModelEngine (Protocol)
├── OllamaEngine
│   - load(model_name, keep_alive=0)   → 모델 메모리 적재
│   - generate(prompt, options) → str  → 추론 실행
│   - unload(model_name)               → 즉시 메모리 해제
│   - is_available() → bool            → Ollama 서버 상태 확인
│
└── ClaudeCliEngine
    - generate(prompt, options) → str  → subprocess 실행
    - is_available() → bool            → which claude 확인
    (load/unload 불필요 — 매 호출마다 subprocess)
```

Summarizer/Translator는 프롬프트 구성 + 응답 파싱만 담당한다. 실제 추론은 주입받은 ModelEngine에 위임한다. 엔진 교체 시 (OllamaEngine → vLLM 등) ModelEngine 구현체만 추가하면 된다.
