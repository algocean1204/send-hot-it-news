## 1. 프로젝트 개요

GeekNews 등 12개 IT/AI 소스에서 뉴스를 **시간당 1회** 자동 수집하고, **로컬 LLM**으로 요약/번역 후 텔레그램 푸시.
macOS 전용. 상시 프로세스 없음. 추가 비용 0원.

## 3. 요약/번역 파이프라인

### 3.1 모델 체인 (순차 로드, 동시 적재 금지)

| 순서 | 모델 | 역할 | RAM |
|------|------|------|-----|
| 1순위 | APEX I-Compact (Qwen3.5-35B-A3B) | 요약 (KO/EN 모두) | ~20GB |
| 조건부 | Kanana-2-30B-A3B (Q4_K_M) | 번역 (EN→KO, 영어 소스만) | ~20GB |
| 최후 | Claude Code CLI (`claude -p --bare`) | 폴백 | 0GB (외부) |

### 3.2 폴백 체인

**발동 조건**: 메모리 부족 (RAM < 26GB) **OR** 모델 오류/타임아웃

```
APEX 시도 → 실패 → Kanana 단독 시도 → 실패 → Claude CLI
```

### 3.3 언어별 처리 흐름

- **한국어 소스** (GeekNews): APEX → 한국어 직접 요약 → 완료
- **영어 소스** (11개): APEX → 영어 요약 → Kanana → 한국어 번역 → 완료

## 5. 필터링 (M3.5 Filter)

### 계층화된 하이브리드 방식

| Tier | 대상 | 전략 | 소스 수 |
|------|------|------|---------|
| Tier 1 | AI랩 블로그 + GitHub releases | 무조건 통과 | 7개 |
| Tier 2 | GeekNews | 큐레이션 신뢰, 전부 통과 | 1개 |
| Tier 3 | Reddit + HN | 업보트 임계값 + 연령 보정 (2시간 미만 = 완화) | 4개 |
| Tier 4 | 전 소스 공통 | 키워드 블랙리스트 (채용/광고/메타/밈) | 전체 |

- 시간당 최대 8건 (MAX_ITEMS_PER_RUN)
- Tier 1+2가 8건 초과 시 그대로 허용 (ALLOW_TIER1_OVERFLOW=true)
- 우선순위 큐: Tier 1 → Tier 2 → Tier 3 (업보트 높은 순)

## 7. Flutter macOS 앱 (풀 대시보드)

### 7.1 확정 기능

- 구독자 관리 (승인/거부/삭제/검색)
- 봇 상태 + 로그 뷰어 + 에러 확인
- 실행 이력 보기
- 날짜별 뉴스 브라우징 (일반/핫뉴스)
- 소스 ON/OFF 토글
- 필터 임계값 조정
- 통계 대시보드
- 헬스체크 (전항목)

### 7.2 통신 방식

- SQLite 직접 읽기/쓰기 (Python과 같은 DB 파일 공유)
- 별도 서버/API 없음

### 7.3 상세 페이지 구성 (7개 화면)

#### 화면 1: 홈 (Overview)
- 봇 상태 (마지막 실행 시각, 성공/실패)
- 오늘 전송 건수
- 최근 에러 요약
- 다음 실행 시간
- 구독자 수 (대기/승인)

#### 화면 2: 날짜별 뉴스
- 30일치 뉴스 날짜별 브라우징
- **일반 뉴스 / 핫뉴스** 탭 분리
- 핫뉴스는 30일 이후에도 **영구 보관** (별도 hot_news 테이블)
- 뉴스 상세: 제목, 요약, 태그, 원문 링크, 파이프라인 경로, 소요 시간

#### 화면 3: 구독자 관리
- 대기(pending) / 승인(approved) / 거부(rejected) 탭
- **검색** 기능 (username, chat_id로 검색)
- 승인 · 거부 · **삭제** 버튼
- 구독자 상세 (신청일, 승인일, 수신 이력)

#### 화면 4: 실행 이력
- 매시 실행 결과 리스트
- 수집/필터/요약/전송 건수
- 총 소요 시간, 모델 로드·추론 각각
- 에러 발생 시 에러 메시지 표시
- 실행 상태: 성공 / 부분 실패 / 실패

#### 화면 5: 오류 로그 + 헬스체크
- **오류 로그 탭**: 시간순 에러 목록, 심각도별 필터
- **헬스체크 탭**: 수동 실행 버튼 + 결과 리포트
  - Ollama 연결 상태 + APEX/Kanana 모델 등록 여부
  - 12개 소스 URL 응답 체크 (RSS 피드 유효, API 응답)
  - Telegram API 상태 (봇 토큰 유효, sendMessage 테스트)
  - SQLite DB 무결성 + 디스크 용량

#### 화면 6: 통계 대시보드
- **소스별 건수 차트**: 일별/주별 소스별 수집·통과·전송 건수
- **파이프라인 성공률**: APEX/Kanana/Claude 각각 성공·실패·폴백 횟수
- **응답 시간 추이**: 회차별 총 소요시간, 모델 로드·추론·전송 각각
- **필터링 효율**: 수집 vs 통과 vs 전송 비율, 블랙리스트 차단 건수

#### 화면 7: 설정 (Settings)
- 소스 ON/OFF 토글 (12개 소스 각각)
- 필터 임계값 조정 (HN min_points, Reddit min_upvotes 등)
- MAX_ITEMS_PER_RUN 조정
- ALLOW_TIER1_OVERFLOW 토글
- YOUNG_THRESHOLD_HOURS 조정

### 7.4 핫뉴스 판단 기준 (복합)

| 방식 | 조건 | 비고 |
|------|------|------|
| 업보트 자동 | HN ≥200, Reddit ≥80 등 소스별 높은 임계값 | 자동 태그 |
| 소스 자동 | AI랩 공식 발표 (Tier 1) + 특정 키워드 (신모델, 버전 릴리즈) | 자동 태그 |
| 수동 지정 | Flutter 앱에서 관리자가 직접 핫뉴스 토글 | 날짜별 뉴스 화면에서 |

- 핫뉴스는 `hot_news` 테이블에 영구 보관 (30일 정리 대상 아님)

## 10. 메인 파이프라인

```
launchd 매시 정각 트리거
        │
        ▼
Memory Guard (psutil → 부족하면 Claude fallback 모드)
        │
        ▼
getUpdates (대기 중인 /start 등 명령어 처리 → subscribers DB 저장)
        │
        ▼
Fetch (12개 소스 병렬 수집)
        │
        ▼
Dedup (SQLite 해시 체크)
        │
        ▼
Filter (Tier 1→2→3 우선순위 큐, 블랙리스트, 최대 8건)
        │
        ▼
신규 0건? ──Yes──→ exit 0
        │ No
        ▼
Language Detect (source-first + lingua fallback)
        │
        ├── KO 아이템 ──┐
        └── EN 아이템 ──┤
                        ▼
        Stage 1: APEX 1회 로드, 전체 요약
                        │
                        ▼
              APEX Unload (keep_alive=0)
                        │
                        ▼
        Stage 2: Kanana 1회 로드, EN만 번역 (조건부)
                        │
                        ▼
              Kanana Unload
                        │
                        ▼
        Telegram 전송 (관리자 + 승인된 구독자)
                        │
                        ▼
              SQLite 커밋 + exit 0
```

## 11. launchd 스케줄

- 16회/일: 09:00 ~ 00:00 매시 정각
- 01:00 ~ 08:00 제외 (수면 시간)
- TZ 이중 고정: plist EnvironmentVariables + Python os.environ['TZ']
- 슬립 복귀 시 ±10분 검증, 벗어나면 skip

## 13. 데이터 관리

### 13.1 SQLite 테이블 (7개)

| 테이블 | 용도 | 보관 기간 |
|--------|------|-----------|
| processed_items | 중복 방지 해시 + 뉴스 데이터 | 30일 자동 정리 |
| hot_news | 핫뉴스 영구 보관 | **영구** |
| subscribers | 구독자 정보 (chat_id, status 등) | 영구 |
| run_history | 매시 실행 결과 기록 | 90일 자동 정리 |
| error_log | 에러 기록 | 30일 자동 정리 |
| filter_config | 소스 ON/OFF + 필터 임계값 | 영구 (설정) |
| health_check_results | 헬스체크 결과 | 7일 자동 정리 |

### 13.2 기타

- 로그: RotatingFileHandler 10MB + 3백업
- .env에 시크릿 (gitignore)
- SQLite WAL 모드 (Python ↔ Flutter 동시 접근 지원)
